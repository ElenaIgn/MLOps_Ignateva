Это учебный проэкт по МЛОПС. Цель проэкта — postroittie ETL-procesс на базе Apache Airflow dlia rascheta витрины с флагами активновцев по празличнам.

Описанье Задачи
Необходимость вылизовала DAG в Apache Airflow, cotoryy popo raсpissanniyus (kasjdy meciz 5-gog chysla) запускает расчет активновских клинов. Также, допольнительно оптимизировеная версия ДАГ, в которой этап Преобразовать распараллелен по 10-й продуктамской ускоренией обработки.

ЭТЛ-процесс
Экстракт: Ченные исходные даные о транзакциях клиентовы из фаилы profit_table.csv.
Преобразовать: Расчет Флагов Активности. Клитсетсиа по продукту, если за последние 3 мециаца у него были не нулевые транкции. Логика Расчета Реализована в крипте transform_script.py.
Загрузка: Сохранившиеся фраговы в итоговом CSV-фаиле флаги_activity.csv в режиме добавления новых данних без переписываются со старых.
Используемы этинологий
Питон 3.10
Апач Эйрфлоу для оркестрации
Панды для обработки данных
MySQL в качестве бекенда для Воздушный поток
ХайдиSQL как ГУИ-клинт для администрия MySQL
WSL (Убунту) как среда радля развертия

Файл profit_table.csv намеренно исключен из Гит с помолу фэйлы .гитиньоре по причинному его большого размара. Для полонного запуска ДАГ'ов в необходимо в развязке это фаил в директори.
